# Code of conduct

We want aikit collaboration to be safe, direct, and useful. This policy applies to repository
discussions, pull requests, issue trackers, community chat, events, and public representation of
the project.

## Expected behavior

- Treat people with respect, including when reviewing mistakes or disagreeing on design.
- Critique code and evidence, not a person's identity, experience, or motives.
- Welcome questions and explain project-specific context without ridicule.
- Protect private information and disclose conflicts of interest.
- Accept maintainer decisions about scope and safety, while remaining free to disagree constructively.
- Prefer precise technical disagreement over sarcasm or personal attacks.

## Unacceptable behavior

Harassment, threats, discriminatory language, sexualized attention, doxxing, sustained disruption,
or retaliation are not tolerated in project spaces or when representing the project.

Also unacceptable:

- posting private vulnerability details in public issues;
- sharing real API keys, private prompts, or customer data in public channels;
- deliberately derailing security or governance discussions with bad-faith arguments.

## Reporting and enforcement

Confidential enforcement reports use the repository's
[private reporting form](https://github.com/matakartal/aikit-runtime/security/advisories/new).
Prefix the report title with `CODE OF CONDUCT` so participant and incident details stay visible only
to the repository owner and security managers.

If the form is unavailable, open a public issue requesting a private contact route **without**
including sensitive incident details.

Maintainers may remove content, restrict participation, or ban contributors when needed to protect
the community. Reports will be handled as confidentially as practical, with conflicts of interest
recused.

## Attribution

This code of conduct is adapted for aikit from community norms commonly used by open-source
projects (including ideas from the [Contributor Covenant](https://www.contributor-covenant.org/)).
It is intentionally short so enforcement remains clear and usable.
